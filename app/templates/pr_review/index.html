PR Review Tool

Automate code review for firm-specific standards and style guidelines

The Problem

Senior developers spend hours reviewing junior code for the same issues: inconsistent naming conventions, missing error handling patterns, non-standard imports. These aren't bugs—they're style and convention violations that every team documents but nobody remembers.

This tool encodes your team's bespoke standards into automated reviews, freeing senior developers to focus on architecture and logic rather than pointing out the same style issues repeatedly.

Your team's standards live in a CLAUDE.md file at the repo root. The tool reads these guidelines and enforces them automatically on every PR.

Example: CLAUDE.md Standards File

Define your team's conventions in plain English. The tool understands context and applies rules intelligently.

CLAUDE.md markdown 
# Code Standards

## Naming Conventions
- Use snake_case for functions and variables
- Use PascalCase for classes
- Prefix private methods with underscore
- Boolean variables should start with is_, has_, or can_

## Error Handling
- Never use bare except clauses
- Always log exceptions before re-raising
- Use custom exception classes for domain errors

## Imports
- Group imports: stdlib, third-party, local
- Use absolute imports, never relative
- Import specific functions, not entire modules

## API Patterns
- All endpoints must have request validation
- Return consistent error response format
- Include request_id in all responses

Before & After Examples

Error Handling Pattern

Before 
def fetch_user(user_id):
    try:
        return db.query(User, user_id)
    except:  # Bare except!
        return None
After 
def fetch_user(user_id):
    try:
        return db.query(User, user_id)
    except DatabaseError as e:
        logger.error(f"Query failed: {e}")
        raise UserFetchError(user_id)

API Response Pattern

Before 
@app.route("/users")
def get_users():
    users = User.query.all()
    return jsonify({
        "users": users
    })
After 
@app.route("/users")
def get_users():
    users = User.query.all()
    return jsonify({
        "request_id": g.request_id,
        "data": {"users": users},
        "error": None
    })
